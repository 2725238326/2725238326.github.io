部署 Hexo 博客的个人心得

Created2025-07-04|Updated2025-07-04|折腾记录
|Post Views:

这几天我花了一些时间，成功把 Hexo 博客部署到自己的服务器上了！
过程中虽然遇到了一些小坑，但也有很多收获。

为什么选择 Hexo

  • 轻量、简洁、开源
  • 写 Markdown 就能发布文章
  • 支持丰富主题，颜值高
  • 生成的是静态网页，部署成本低

踩过的坑

  • glibc 版本太旧，导致 Node.js 无法运行
  • Git 推送失败，要配置正确的 SSH 公钥或密码
  • Nginx 初始配置不对，访问首页出现 404
  • YAML 配置文件缩进出错，Hexo 直接崩了！

我的部署流程简述

  1. 安装 Node.js、Hexo、Git 等环境
  2. 初始化 Hexo 项目
  3. 配置 Nginx 指向 Hexo 的 public 目录
  4. 使用 hexo g 生成静态文件，上传到服务器
  5. 搭配 Git 仓库 + Webhook 实现一键自动部署 ✨

收获与建议

  • 备份很重要：系统、配置文件、博客源码都要有备份
  • 文档是朋友：多看官方文档，少走弯路
  • 自动化部署：一次设置，终身省心

如果你也在考虑搭建博客，希望这篇文章能帮你少踩几个坑！
